随笔--粗谈硕士那几年

Posted on January 9, 2020 本文总阅读量次

​ 算是17年7月入得学，快满三年，我的同门专硕即将离校。

​ 在17年3月份复试通过后，本以为能好好玩个暑假，奈何导师负责，常常催促着我来校报到，心里肯定是不愿意的，但脑子里的固有思维还是想给导师留个好印象。写这篇记事时，已是2020一月有余。此刻还是清晰的记得当时是17年的7月6日到校的，拖着个大行李箱，顶着个大太阳，大汗淋漓，时间记得这么清楚也可能是因为那天是我的阴历生日，hhh。

​ 在宿舍睡得第一天，舍友都迟迟未归，我的几个舍友都是同门，后来谈论起，他们都在实验室学习。他们的整体业务能力和工程素质都是甩我好几条街的，康哥、全林、锋神在本科期间都是参加过无数比赛的人，其中不乏肱骨之才，据我所知，康哥还是小车的队长，甚是厉害。

​ 导师实验室的工作基本是围绕着图像展开，研一那会儿，主要业务集中在嵌入式方向，不过在当时也算是“朝代末期”了。当时的研三师兄都是嵌入式能手，和公司合作的项目都是他们做的，例如KVM视频编码、海思3516相关开发等。丰富的嵌入式经验也让他们在找到了很多很好的offer，最后三人去了华为，还有大师姐去了道富银行，也是滋润。

​ 除了平时承接的项目外，当时导师的学术工作，主要集中在视频编解码这块（现在是围绕着机器人做研究开发了，相当厉害），我算是最后接这一棒的人。HEVC自2013年推出，我的前人已经做了很多研究，导师也曾说，现在都是硬编码，再做下去，工程上已然没有实际价值，学术上仍有一定的意义，主要还是集中的深度图编码上，当时我的师姐，也是沿着这一方向，中了一篇EI和国内A类论文，相当厉害。

​ 我在编解码上的研究不算很深，源码看过一段时间，基本能能走通它们的流程（H264、HEVC），不过要让我加、改逻辑，还是很吃力的。除了一些视频编码框架，我还研究过大半年的JPEG编码，当时是导师接的一个项目，听说价格不菲，依托着光纤，实现低延时视觉无损压缩，当时也做了很长一段时间，掌握了JPEG的编码流程，8*8分块、颜色格式转换、DCT变换、量化、熵编码等，做了不少数据统计。在熟悉了之后也要做出一定的改进，至于怎么改进，就不方便写出来了，毕竟是导师的项目，目前也还没做完，我也不想做了，只想躺倒6月准时毕业。

​ 导师在工程上对我试错了之后，我是真的不是那块料，随后给我安排毕业中的大事：论文。起初的方向也是编解码，但是用深度学习或机器学习来做，当时的点子的有如下这x些：

  1. 用机器学习对PU、CU进行模式决策，加速编码

  2. 通过深度学习new一个端到端的框架，实现图像的编解码

由于第一点，算是很老的点子了，所以工作基本围绕着第二点展开。

​ 当时实验室已经购入了两台服务器（现在已有四台，实验室冬天的不用开空调了），配置也相当豪华，一台Titan，另一台是两张1080显卡，不过我都没机会去用，现在有机会去使用了，但是真的难！！！此前老师也跟我沟通过，大致内容就是这个方向你觉得OK吗，我也表达了我的能力可能不足，这一点上导师给我了很大的信心，说实验室这么多大牛，在环境搭建上都能帮得上。

​ 那就干吧。

​ 看博客的同时，在github上找源码，当时很抵触英文论文，但是新的、 好的东西都是它们在搞，不过这两年国内的深度学习的工作走在了国际前沿：AI四小龙、海康、滴滴、百度等等大公司，都有不俗的表现。

​ github上不乏很多的项目，clone下来后，第一步就是要如何跑通。但就是这第一步，把我难了好长一段时间，在本文中表达对锋神的强烈感谢，当时都是拜托它，一步一步把网络跑起来的。现在回想起，其实也不难（hhh马后炮）。

  1. clone，这里一般网速都成功，不过推荐一个工具DOWNGIT，挺好用。
  2. requirement的安装，可能有些项目中有，有些没有，这个在代码调试过程中会有报错信息，pip install 可以解决
  3. dataset的下载，这一点着实头疼，百度网盘也还好就网速慢点，可是谷歌云盘就很下载成功，这里推荐梯子+IDM，开全局下载，一般能成功：chrome先下载，在复制连接到IDM中去。
  4. ./train.sh或者python train.py（python版本会有不同）
  5. test
  6. 论文工作：进行指标评价，其中指标评价的工具，github上也会有，你去给原作者发issue即可。

大致就是以上几步，另外推荐使用virtualenv（workon env相当方便）或者pyenv（这个我没用过），不要用anaconda，不方便。

然而，你会跑，发论文肯定不行。

那就需要看源码和看论文，双管齐下。这里讲一下我的路子。

  1. 检索、阅读阶段。首先是search，推荐谷歌学术或者直接谷歌搜索，这样会出来很多英文论文，进行文献的下载，结合着blog（不局限于各大博客，私人写得好的也很多）进行阅读。
  2. 集中看几篇，选择依据：时间新的、效果好的、代码有的或者代码全面的（方便修改）
  3. 同领域的进行论文检索，这一步，是为了引进同领域先进的网络架构或者模块，为我所用。
  4. 实验部分，考验你的代码功底，推荐先去看一遍python莫凡的视频，我研一来的那会儿，生蚝同学天天做我旁边看，现在已是一名炼丹师，就职于海康算法研究院。
  5. 写作部分，先照葫芦画瓢。这里导师的细致，让我敬佩，我的论文、专利都改了十稿以上才发表。
  6. 审稿阶段，来来回回的回复审稿专家的意见，一定要诚心诚意，要求加实验的，一定要加上。

上面讲了一下我的写论文之旅，不过我最后发的论文并不是基于深度学习的图像编码，这一领域实在不好做并且我感觉没啥实际价值，速度慢，效果差，顶多只能算是创新，可这创新，谷歌已经帮你做玩了，并且只是低码率的情况下，会比JPEG好很多，其余全被吊打。

​ 后来的我选择的是对象级的边缘检测方向，具体的可检索我的论文，发表在计算机科学上。

写论文之后，正式全身心的投入到秋招中去，面了无数家公司，投递职位主要有这些：围绕着C/C++、视频编解码（可以带点深度学习的）、银行、移动等，也拿到一些offer：舜宇、同花顺、海康、大华、工商银行杭研、杭州银行。近期报了浙江二月份的省考，希望一切顺利。

​ 以上算是已经发生的研究生期间的几件大事吧。当然有很多的小事没放进去。总结我研究生期间的技术栈吧：C/C++、Opencv、Linux、Python等，目前在学习MySQL、Java。

大论文也在筹备阶段，不过还没做完，有一个应用的做不出来，不知道最后咋办……

​ 大前天与同门拍了毕业照，祝愿他们前程似锦（比我多挣半年的工资，羡慕）。

​ 今天的天又阴了。就先写到这儿吧。
